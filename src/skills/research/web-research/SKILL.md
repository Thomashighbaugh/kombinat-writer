# Web Research

## Search Strategy Formulation

Construct searches using Boolean operators (`AND`, `OR`, `NOT`) to combine or exclude terms. Use `AND` to narrow results, `OR` to capture synonyms (`"climate change" OR "global warming"`), and `NOT` to exclude irrelevant contexts. Apply domain restriction (`site:.edu`, `site:.gov`, `site:.org`) and date filtering (`after:2022`, `before:2020`) to kombinate results. Use filetype filtering (`filetype:pdf`) to locate reports and papers.

For complex topics, decompose the research question into 3–5 atomic queries, each targeting a distinct facet: historical context, current data, expert opinion, counter-arguments, and practical application. Use phrase matching (`"exact phrase"`) for proper nouns, titles, and legislation names.

## Query Refinement

When results are too broad, narrow by adding domain-specific terms, excluding common terms with `NOT`, or combining facets with `AND`. When too narrow, relax constraints systematically: remove date filters first, then domain restrictions, then phrase matching, then the most specific term. Log each query and search engine used to document the search trail and prevent redundancy. Use engine-specific operators: Google supports `AROUND(N)` for proximity; DuckDuckGo supports `!` bang syntax for site redirection.

## Source Credibility Quick-Assessment

Evaluate every source on five axes. **Authority**: author credentials and publication outlet reputation. **Currency**: publication date and revision history. **Purpose**: informative, persuasive, commercial, or satirical — look for disclosure and advocacy framing. **Evidence quality**: citations, data sources, peer review status. **Bias signal**: loaded language, omitted counter-arguments, emotional framing.

Apply the SIFT method: **Stop** before engaging deeply — **Investigate** the source's reputation — **Find** trusted independent corroboration — **Trace** claims to their original context rather than relying on secondary summaries.

## Research Workflow

Execute the closed-loop sequence: **Question** → **Search** → **Evaluate** → **Extract** → **File**. Return to Question when results are insufficient or reveal a need for reframing. Cycle for each sub-question. Maintain a research log tracking question → query → source → extraction for full auditability.

## Avoiding Filter Bubbles and Confirmation Bias

Search across ideological spectra for contested topics — use sources from different geographic regions, political orientations, and methodological traditions. Use search engines with differing ranking algorithms (Google, DuckDuckGo, Bing, marginalia.nu) to bypass personalisation. Use incognito sessions to minimise algorithmic curation.

Before committing to a finding, articulate the strongest counter-argument, then search for evidence supporting it. Maintain a "disconfirmed assumptions" document to counter the tendency to privilege confirming evidence over contradictory data.

## Tracking Search Provenance

Record for every session: the exact query string, search engine, timestamp, and for each result, the sourcing decision (included, excluded, pending, deferred) with rationale. This trail enables reconstruction of the research path, supports citation transparency, and prevents redundant searching. A structured log with columns for query, source URL, decision, and rationale is recommended.

## Synthesising Multiple Sources

Build a synthesis matrix after extracting from multiple sources: rows are sources, columns are themes, cells contain key findings or quotes. Identify convergence (multiple sources supporting the same finding), divergence (contradictory findings), and gaps (insufficient coverage). Grade each finding by evidentiary weight — number and quality of supporting sources. Synthesis, not accumulation, is the goal of research.
